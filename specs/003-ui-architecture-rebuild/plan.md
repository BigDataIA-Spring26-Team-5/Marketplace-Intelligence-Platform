## Phase 1: Design

### UI Flow Mapping (5 Wizard Steps → 7 Pipeline Nodes)

| Wizard Step | Pipeline Nodes | Agent | HITL Gate |
|------------|----------------|-------|-----------|
| Step 1: Select Source | load_source | — | — |
| Step 2: Schema Analysis | analyze_schema (Agent 1), critique_schema (Agent 2) | Agent 1, Agent 2 | Gate 1 (after Agent 1) |
| Step 3: Pipeline Planning | check_registry, plan_sequence (Agent 3) | Agent 3 | — |
| Step 4: Execution | run_pipeline, save_output | — | Gate 3 (if quarantine) |
| Step 5: Results | — | — | — |

### Component Updates Required

1. **render_agent_header()**: New function to display Agent 1/2/3 with role labels
2. **render_sampling_stats()**: New function to display sampling strategy from Agent 1
3. **render_confidence_badges()**: Update to show High/Medium/Low badges on gaps
4. **render_extraction_only_flag()**: New function for safety constraint columns
5. **Step bar update**: Label steps with agent names where applicable

### Key UI Elements

- **Agent 1 (Orchestrator)**: "Schema Analysis" - shows sampling stats, confidence scores
- **Agent 2 (Critic)**: "Schema Validation" - shows revision notes, corrections
- **Agent 3 (Sequence Planner)**: "Execution Order" - shows block sequence with reasoning
- **HITL Gate 1**: "Approve Mapping & Continue" / "Exclude Column" / "Abort Ingestion"
- **HITL Gate 3**: "Accept Quarantine (exclude)" / "Override: Include All"
- **Safety columns**: allergens, is_organic, dietary_tags show "EXTRACTION-ONLY" badge
