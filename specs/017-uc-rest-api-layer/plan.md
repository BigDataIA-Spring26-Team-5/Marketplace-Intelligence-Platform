# Implementation Plan: UC REST API Layer

**Branch**: `017-uc-rest-api-layer` | **Date**: 2026-04-24 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/017-uc-rest-api-layer/spec.md`

## Summary

Add a structured REST API layer (`src/api/`) exposing UC1 pipeline control, UC2 observability queries, UC3 hybrid search, and UC4 recommendations as proper HTTP endpoints with FastAPI routers. The existing MCP server (`mcp_server.py` on port 8001) is preserved unchanged; the new API runs on port 8002 and mounts the MCP app as a sub-application at `/mcp`. Pipeline runs execute asynchronously — `POST /pipeline/runs` returns a `run_id` immediately; the existing `CheckpointManager` SQLite store is reused for status persistence.

## Technical Context

**Language/Version**: Python 3.11 (Poetry)
**Primary Dependencies**: FastAPI (already in stack via mcp_server.py), Uvicorn, Pydantic v2
**Storage**: SQLite via CheckpointManager (run state); Postgres (observability queries); Redis + SQLite fallback (cache)
**Testing**: pytest (existing suite); new `tests/api/` for contract and integration tests
**Target Platform**: Linux server (same host as pipeline)
**Project Type**: web-service (REST API)
**Performance Goals**: Status poll < 50ms; search results < 2s (index warm); run submission < 200ms
**Constraints**: Must not break existing CLI, Streamlit, or MCP server invocation paths
**Scale/Scope**: Single-server; same process as pipeline; no worker queue for phase 1

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-checked after Phase 1 design.*

- **Schema impact**: None. API layer reads `config/schemas/<domain>_schema.json` via `GET /ops/schema/{domain}` but never writes it. Schema writes remain sole domain of Agent 1.
- **Agent architecture**: Unaffected. `POST /pipeline/runs` calls `graph.invoke()` through the existing graph — no new agent responsibilities introduced.
- **YAML-only transforms**: Unaffected. The API is a call-through layer; no transforms execute in the API routers.
- **HITL gates**: Not bypassed. Streamlit gates remain the only HITL path. API runs execute the full graph including `critique_schema` when `with_critic=true`.
- **Enrichment safety boundary**: Not touched. API does not expose safety field inference; `SearchHit` and `RecHit` are read-only catalog views.
- **DQ scoring / quarantine**: Preserved. `RunResult` exposes `dq_score_pre`, `dq_score_post`, `rows_quarantined` from the existing audit trail.
- **Documentation updates**: `CLAUDE.md` Active Technologies section, `ENDPOINTS.md`, and this plan require updates.

**Constitution violations**: None.

## Implementation Phases

### Phase A — Foundation (do first)
1. `src/api/main.py` — bare FastAPI app, `/health` endpoint, MCP sub-app mount
2. `src/api/dependencies.py` — singletons for CacheClient, CheckpointManager
3. `src/api/models/` — all Pydantic models from data-model.md
4. Verify `poetry run uvicorn src.api.main:app --port 8002` starts and `/health` returns 200

### Phase B — Pipeline router
5. `src/api/routers/pipeline.py` — submit (BackgroundTasks), status, result, resume, cancel stub
6. `tests/api/test_pipeline.py` — submit→poll→result happy path with a small CSV

### Phase C — Observability router
7. `src/api/routers/observability.py` — wrap `_pg_query`/`_prom_query` helpers from mcp_server; add pagination + filters
8. `tests/api/test_observability.py` — query against seeded run logs

### Phase D — Search router
9. `src/api/routers/search.py` — call `HybridSearch.search()`, guard with `is_ready()`
10. `tests/api/test_search.py` — 503 when not ready; 200 with results when corpus present

### Phase E — Recommendations router
11. `src/api/routers/recommendations.py` — call `Recommender.also_bought()`/`you_might_like()`, guard with `is_ready()`
12. `tests/api/test_recommendations.py` — 503 when graph not built; 404 for unknown product

### Phase F — Ops router
13. `src/api/routers/ops.py` — cache stats/flush via CacheClient, schema from JSON file
14. `tests/api/test_ops.py` — flush requires confirm=True; schema 404 for unknown domain

### Phase G — Docs + polish
15. Update `ENDPOINTS.md` with new port 8002 endpoints
16. Update `CLAUDE.md` Active Technologies section
17. Verify existing MCP tools still reachable at `localhost:8001` and `localhost:8002/mcp`
