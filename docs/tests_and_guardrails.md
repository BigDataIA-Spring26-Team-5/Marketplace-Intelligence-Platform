# Tests & Guardrails Documentation

## How to Run Tests

```bash
# From the project root directory, open a terminal and run:
poetry run pytest tests/unit_tests.py -v

# Run only a specific test section:
poetry run pytest tests/unit_tests.py -v -k "TestColumnSpec"
poetry run pytest tests/unit_tests.py -v -k "Hallucination"
poetry run pytest tests/unit_tests.py -v -k "HITL"

# Run with short traceback on failures:
poetry run pytest tests/unit_tests.py -v --tb=short

# If poetry virtualenv is not set up, use pip-installed pytest:
python -m pytest tests/unit_tests.py -v
```

## Guardrails Implemented (`src/agents/guardrails.py`)

### Overview

The guardrails module provides validation for all 4 LLM call sites in the pipeline:

| # | Agent | File | Function |
|---|-------|------|----------|
| 1 | Schema Analyzer (Agent 1) | `src/agents/orchestrator.py` | `analyze_schema_node()` |
| 2 | Critic (Agent 2) | `src/agents/critic.py` | `critique_schema_node()` |
| 3 | Sequence Planner (Agent 3) | `src/agents/graph.py` | `plan_sequence_node()` |
| 4 | LLM Enrichment (S3) | `src/enrichment/llm_tier.py` | `llm_enrich()` |

---

### 1. Input Guardrails (Pre-LLM Call)

Validated **before** the LLM is called — prevents wasting tokens on invalid inputs.

| Guardrail | Call Site | What It Checks |
|-----------|-----------|----------------|
| Empty source schema | Schema Analysis | source_schema must have actual columns |
| Empty unified schema | Schema Analysis | unified_schema must be non-empty |
| Only `__meta__` columns | Schema Analysis | Must have real data columns, not just metadata |
| Oversized payload warning | Schema Analysis | Warns if schema >100K chars (token waste) |
| Empty mapping + operations | Critic | At least one must be non-empty |
| Empty source profile | Critic | Critic needs data to verify against |
| Column overlap detection | Critic | Warns if same column in both mapping and operations |
| Empty blocks_metadata | Sequence Planner | Must have blocks to sequence |
| Empty domain | Sequence Planner | Planner needs domain context |
| Empty rows | Enrichment | Must have rows to enrich |
| Batch size range | Enrichment | Warns if batch_size outside [1, 100] |
| Missing product_name | Enrichment | Warns if rows lack product_name field |

---

### 2. Output Guardrails (Post-LLM Call)

Validated **after** parsing the LLM response — catches hallucinations and structural errors.

#### 2a. Response-Level Checks (All Call Sites)

| Guardrail | Severity | Detection |
|-----------|----------|-----------|
| Response >200K chars | **Error** | Runaway generation / infinite loop |
| Non-JSON response | **Error** | Supports markdown-fenced fallback |
| Prompt echo-back | Warning | LLM confused — echoing instructions |

#### 2b. Schema Analysis Output (Agent 1)

| Guardrail | Severity | Detection |
|-----------|----------|-----------|
| Hallucinated source column in `column_mapping` | **Error** | Column doesn't exist in source data |
| Hallucinated source column in `operations[].source_column` | **Error** | Same check for CAST/FORMAT/DERIVE/SPLIT |
| Mapping to enrichment column | **Error** | Enrichment cols filled by downstream blocks |
| Mapping to computed column | **Error** | Computed cols generated by pipeline |
| Invalid primitive name | **Error** | Only 9 allowed: RENAME, CAST, FORMAT, DELETE, ADD, SPLIT, UNIFY, DERIVE, ENRICH_ALIAS |
| Invalid action per primitive | **Error** | Each primitive has a fixed set of valid actions |
| Targeting safety column | **Error** | allergens/is_organic/dietary_tags are extraction-only |
| Targeting computed column | **Error** | dq_score_pre/post/delta are pipeline-generated |
| Operations count >500 | **Error** | Likely hallucination explosion |
| Duplicate (primitive, target) pair | Warning | Same operation repeated — loop signal |

#### 2c. Critic Output (Agent 2)

| Guardrail | Severity | Detection |
|-----------|----------|-----------|
| 3x more operations than input | **Error** | Disproportionate output = hallucination |
| Invalid primitive in revised_operations | **Error** | Must use valid taxonomy |
| Target column not in unified schema | Warning | Critic invented a column |
| Missing `rule` field in critique_notes | Warning | Structural validation |
| Non-list revised_operations | **Error** | Response structure broken |

#### 2d. Sequence Planner Output (Agent 3)

| Guardrail | Severity | Detection |
|-----------|----------|-----------|
| Hallucinated blocks (not in registry) | **Error** | Unknown block names |
| `dq_score_pre` not first | **Error** | Ordering constraint |
| `dq_score_post` not last | **Error** | Ordering constraint |
| Normalization after dedup | **Error** | Must normalize before dedup |
| `extract_allergens` after `llm_enrich` | **Error** | Allergens must be extracted first |
| Missing blocks | Warning | Will be auto-appended |

#### 2e. Enrichment Output (S3)

| Guardrail | Severity | Detection |
|-----------|----------|-----------|
| Category not in VALID_CATEGORIES | Warning | Hallucinated category (flagged for review) |
| Safety column present (allergens, is_organic, dietary_tags) | **Error** | Strips value — LLM must never infer these |
| idx out of bounds | Warning | Skipped — hallucinated index |
| Invalid idx type | Warning | Skipped |
| Non-string primary_category | Warning | Skipped |
| `null` category | Allowed | Preferred over hallucination |

---

### 3. HITL Threshold Guardrails

Flags high-impact outputs for human review (does not block, but requires approval).

| Threshold | Default Value | Trigger Condition |
|-----------|---------------|-------------------|
| Operation count | >15 | Complex transformation warrants human review |
| Unresolvable columns | >5 | Significant data gaps need human decision |
| Average confidence | <0.5 | Low-confidence mappings need verification |
| Enrichment batch size | >50 | Large batches should be spot-checked |

**Data classes:**
- `HITLFlag` — captures `triggered`, `reason`, `threshold_name`, `actual_value`, `threshold_value`
- `GuardrailAudit` — full audit trail with `call_site`, `model_version`, `checks_performed`, `hitl_flags`, `elapsed_ms`, `timestamp`

---

### 4. Numerical Clamping

Prevents out-of-range values from propagating through the pipeline.

| Function | Valid Range | Default on Invalid Input |
|----------|------------|--------------------------|
| `validate_confidence_score(score)` | [0.0, 1.0] | 0.0 |
| `validate_dq_score(score)` | [0.0, 100.0] | 0.0 |
| `validate_risk_score(score)` | [1, 5] | 3 |

---

### 5. Production Entry Point

```python
from src.agents.guardrails import run_guardrails_with_audit

result, audit = run_guardrails_with_audit(
    call_site="schema_analysis",
    raw_response=raw_llm_response,
    parsed_result=parsed_json,
    model_version="deepseek/deepseek-chat",
    confidence_scores=[0.9, 0.85, ...],  # optional
    source_columns={"col1", "col2"},
    unified_columns={"product_name", ...},
    enrichment_columns={"primary_category", ...},
    computed_columns={"dq_score_pre", ...},
)

if not result:
    # LLM output is invalid — retry or fallback
    print(result.errors)

if audit.requires_human_review:
    # Flag for human approval
    for flag in audit.hitl_flags:
        print(f"HITL: {flag.reason}")
```
